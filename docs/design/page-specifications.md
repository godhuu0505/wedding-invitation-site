# ページ仕様

## ページ構成概要

| ページ名 | URL | 機能概要 | 優先度 |
|---------|-----|---------|-------|
| ホーム | `/` | メインビジュアル・結婚式概要・ナビゲーション | 最高 |
| プロフィール | `/profile` | 新郎新婦の紹介・写真・馴れ初め | 高 |
| 式場案内 | `/venue` | 会場情報・Google Maps・アクセス方法 | 最高 |
| 出欠確認 | `/rsvp` | 出欠フォーム・同伴者情報・メッセージ | 最高 |
| サンクス | `/thank-you` | フォーム送信完了・お礼メッセージ | 高 |
| 管理画面 | `/admin` | 出欠状況確認・ゲスト管理（認証付き） | 中 |

## ホームページ (`/`)

### 目的
結婚式の第一印象とナビゲーションハブ

### 表示内容
- **メインビジュアル**: 新郎新婦の写真・結婚式タイトル
- **基本情報**: 
  - 開催日時: 2025年11月3日（文化の日）
  - 会場: 東京都内の式場（詳細は式場案内ページで）
  - 挙式・披露宴の時間帯
- **ナビゲーション**: 各ページへのリンク
- **重要なお知らせ**: 更新情報・注意事項

### 技術実装
- ファイル: `app/page.tsx`
- レスポンシブ画像表示（Next.js Image コンポーネント）
- アニメーション効果（Framer Motion 予定）

### UI/UX要件
- ファーストビューで結婚式の雰囲気が伝わること
- モバイルでも見やすいタイポグラフィ
- 直感的なナビゲーション配置

## プロフィールページ (`/profile`)

### 目的
新郎新婦の人となりを紹介

### 表示内容
- **新郎プロフィール**: 名前・年齢・出身地・趣味・一言
- **新婦プロフィール**: 名前・年齢・出身地・趣味・一言
- **馴れ初め**: 出会いから結婚に至るストーリー
- **写真ギャラリー**: カップル写真（3-5枚）

### 技術実装
- 画像最適化（WebP対応）
- レスポンシブレイアウト
- 写真のLazy Loading

### UI/UX要件
- 親しみやすい雰囲気
- 写真とテキストのバランス
- スクロール体験の最適化

## 式場案内ページ (`/venue`)

### 目的
会場情報とアクセス方法の提供

### 表示内容
- **会場基本情報**: 名称・住所・電話番号
- **Google Maps 埋め込み**: インタラクティブな地図表示
- **アクセス方法**: 
  - 電車でのアクセス（最寄り駅からの徒歩ルート）
  - 車でのアクセス（駐車場情報含む）
- **会場写真**: 外観・内観の写真
- **注意事項**: 会場特有のルール・ドレスコード

### 技術実装
- Google Maps JavaScript API
- カスタムマーカーとインフォウィンドウ
- ルート検索機能

### UI/UX要件
- 地図の見やすさ重視
- アクセス情報の分かりやすい整理
- モバイルでの地図操作性

## 出欠確認ページ (`/rsvp`)

### 目的
包括的な出欠確認とゲスト情報収集

### 入力項目
- **基本情報**:
  - お名前（必須、テキスト入力）
  - ふりがな（必須、ひらがな）
  - 連絡先（メールアドレス、必須）
- **出欠情報**:
  - 出欠（必須、ラジオボタン: "出席" / "欠席"）
  - 同伴者数（数値、0-5人）
  - 同伴者名（テキストエリア、同伴者がいる場合）
- **その他**:
  - 食物アレルギー（テキストエリア、任意）
  - お祝いメッセージ（テキストエリア、任意）
  - 連絡事項（テキストエリア、任意）

### バリデーション
- 名前・ふりがな・メールアドレス・出欠: 必須
- メールアドレス: 形式チェック
- 同伴者数: 0-5の範囲制限
- 文字数制限: メッセージ類は500文字以内

### 動作フロー
1. フォーム入力・リアルタイムバリデーション
2. 送信前確認モーダル表示
3. Firebase Firestoreへのデータ保存
4. 送信完了でサンクスページに遷移
5. 管理者への通知メール送信（Cloud Functions）

### 技術実装
- コンポーネント: `components/RSVPForm.tsx`
- バリデーション: React Hook Form + Yup
- APIエンドポイント: `app/rsvp/submit.ts`
- 状態管理: Zustand（予定）

### UI/UX要件
- ステップバイステップの入力体験
- エラーメッセージの分かりやすい表示
- 入力進捗の可視化

## サンクスページ (`/thank-you`)

### 目的
送信完了の確認とお礼

### 表示内容
- 送信完了メッセージ
- 今後の流れの説明
- 連絡先情報
- ホームページへの戻りリンク

### UI/UX要件
- 安心感を与えるデザイン
- 次のアクションが明確
- 感謝の気持ちが伝わる文言

## 管理画面 (`/admin`)

### 認証システム
- Firebase Authentication（メール・パスワード）
- 管理者アカウントは事前登録制
- セッション管理・自動ログアウト

### 管理機能
- **出欠一覧**: 全ゲストの出欠状況表示
- **統計情報**: 出席者数・欠席者数・同伴者総数
- **検索・フィルタ**: 名前検索・出欠状況での絞り込み
- **データエクスポート**: CSV形式でのダウンロード
- **メッセージ確認**: ゲストからのお祝いメッセージ一覧

### UI/UX要件
- ダッシュボード形式の情報整理
- データテーブルの見やすさ
- 管理作業の効率性重視
